# Live Trading Tells

Use this file when live orderbook behavior or event sequence is itself giving information.

A tell is not deterministic. It is a clue that should change how you price execution, path risk, or urgency.

## 1. Full-size fill from one side
- You get filled for your whole resting size at once.
- Often means the counterparty may know something you do not.
- Recheck news / rules / event path before assuming the fill was just lucky.

## 2. Book moves before you finish thinking
- Price runs away while you wait for ideal passive fills.
- Sometimes means edge is decaying fast enough that pure maker discipline is no longer optimal.

## 3. Informed pump implies rest of field weaker
- In field markets, concentrated informed-looking buying is itself negative information for untouched names.

## 4. Setup-cue spike without live confirmation
- Price jumps on room/setup shorthand before the event confirms that the path is really opening or closing.

## 5. Late path stays too cheap after silence
- Word misses opening remarks, but the market kills it too hard even though Q&A / detour path is still alive.

## 6. Garbage-time broadcast drift
- Once a game script dies, commentary language can become structurally different from earlier phases.

## 7. Logistics signal before performance
- Stage/stadium/rehearsal clues can matter, but the market often prices them too absolutely.

## Use rule
If a tell is the main reason you like a trade, say so explicitly.
Do not hide live-path reasoning behind a fake static fair-value number.
