# PMT Playbook

## Core principle
Edge in mention markets comes from price, setup repricing, execution, event structure, and live market microstructure more than from raw binary guessing.

## Recommended reasoning order

1. **Archetype**
   - What kind of market is this?
   - Recurring announcer market, Trump live Q&A market, briefing market, named policy announcement, performer field market, bond-like market, or something else?

2. **Format**
   - What event format is driving the path?
   - Are prepared remarks and Q&A separate distributions?
   - Is the market overreacting to setup shorthand instead of actual path structure?

3. **Speaker / recurrence / uncertainty**
   - Is this a vibes-heavy speaker, a recurring/modelable speaker, or a thin-history speaker?
   - Should you trust comparables, segment them, or heavily widen the confidence band?

4. **Pricing signal / crowd mistake**
   - What is the market likely getting wrong?
   - Pooled historicals? setup-label overreaction? field misread? copy-trader premium? phase-blind pricing? direct-name underpricing? thin-history fake precision?

5. **Fair value**
   - Only now estimate rough fair value.
   - Price the specific path, not just the theme.

6. **Execution**
   - Decide whether passive entry is realistic.
   - If the edge is decaying, decide whether taker execution is justified.
   - Treat live orderbook behavior as information, not just friction.

7. **Sizing**
   - Scale size to setup clarity, recurrence, fill quality, bankroll opportunity cost, and dispute sensitivity.
   - Do not size a weird one-off like a recurring edge just because the narrative sounds good.

8. **Risk / invalidation**
   - Identify the main path risk, dispute risk, and no-trade condition.

## Core sections

### Price first
- Treat the same strike at different prices as different trades.
- A plausible mention can still be overpriced.
- Compare to fair value, not to vibes alone.
- The crowd can be “directionally right” and still leave edge on the other side if price overshoots.

### Execution
- Limit orders are default.
- Maker-style execution is an edge in wide-spread mention markets.
- Aggressive taking is justified only when edge is decaying in real time.
- Scale in instead of exposing one giant visible order.
- Realized EV depends on fill quality, queue position, and how fast the book is moving.
- Do not confuse a good model with good execution.

### Setup repricing
- Rebuild fair value if the event frame changes.
- Use venue, event type, guest mix, and Q&A likelihood as core variables.
- Sparse formats deserve wider confidence bands.
- Setup labels like open press / closed press are useful cues, but they are not the whole model.

### Timing / phase logic
- Late entry is often superior when information improves near start.
- Event start changes liquidity conditions.
- Live sequence matters in partially triggered or dispute-prone markets.
- Some strikes are prepared-remarks strikes; some are Q&A strikes; some only become live in late detours or garbage-time chatter.
- “Not said yet” is not always the same as “dead.”

### Relative value
- Compare neighboring strikes.
- Compare aggregate and component markets.
- Use similar events as anchors, then adjust for current context.
- In field markets, repricing one name should usually reprice the rest of the field too.

### Risk discipline
- Avoid fake precision.
- Respect dispute risk.
- Do not confuse thesis correctness with trade quality.
- Do not confuse a nice narrative with a repeatable archetype.
- Respect capital allocation: a decent bond-like trade can still be worse than keeping bankroll available for faster recurring edge.
